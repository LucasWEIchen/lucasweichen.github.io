ROS Catkin_make errors summary

2018年03月04日 | ROS CATKIN_MAKE SLAM

  1. reinstall ros will give you a error saying apt-get couldn’t install ros-kinetic-simulator. This is most likely because you still got gazebo installed on you Ubuntu. uninstall it and try install ROS again.

  2. catkin_make some times give your permission error. This can be addressed by use chmod -x . Or oddly, it didn’t work for me. I solve this by using Ubuntu GUI and actually right click on that .cfg file and change the permission in the drop list. I still do not get it, why my cmooand line not working?

  3. delete both devel and build folder in your workspace can sometimes help to solve problem. But, for me, it some times cause more problems than it fixed.
